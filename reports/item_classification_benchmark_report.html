Item Classification Benchmark Report

这份报告解释当前 item-level 分类 benchmark 是怎么设计的、是否包含大类和小类、 当前实际评估了什么，以及 legacy embedding 与 Harrier embedding 的结果。

Branch: llm-memory Dataset: benchmarks/item_classification_benchmark.csv Scope: item main_category accuracy Report date: 2026-07-02

Short Answer

Benchmark 数据集里同时有大类和小类 gold label： gold_main_category 和 gold_sub_category。

但是当前 benchmark 的准确率只评估大类，也就是 main_category。小类现在只是先保存在数据集中，方便下一阶段扩展。

Dataset Design

120 gold-labeled item rows
12 main categories covered
10 items per covered main category
main only current scoring target

每条 benchmark row 表示一个已经 OCR/receipt extraction 完成后的 item。字段如下： 

id, merchant, raw_name, quantity, amount, currency, gold_main_category, gold_sub_category

当前 benchmark 覆盖了 12 个 main categories，每个 10 条样本： 

{
  "groceries": 10,

完整 taxonomy 有 18 个 main categories。当前 benchmark 还没有覆盖： travel, housing_utilities, children_family, gifts_donations, insurance, other。

Is It Main Category or Subcategory?

Level Exists In Dataset? Currently Scored? Reason
gold_main_category Yes Yes All compared models can produce or be mapped to a main-category prediction.
gold_sub_category Yes No Legacy embedding and Harrier zero-shot currently only output main categories. Subcategory scoring would be unfair unless every model outputs subcategory.

Example row: 

{
  "merchant": "REWE",
  "raw_name": "ZAHNCREME",
  "gold_main_category": "personal_care",
  "gold_sub_category": "personal_care.dental_care"
}

Models Compared

Model Input Method Output Evaluated
Legacy embedding merchant only Existing SentenceTransformer nearest-centroid classifier from dataset/centroids.json, then mapped from old transaction categories to taxonomy main categories. main_category
Harrier 0.6B merchant + raw_name + amount microsoft/harrier-oss-v1-0.6b as zero-shot embedding classifier against taxonomy main-category descriptions, with merchant-focused candidate narrowing. main_category

What This Means

  • The current benchmark is an item-level benchmark, but it only scores main-category accuracy for now.
  • The gold dataset already contains subcategory labels, so subcategory evaluation can be added later without rewriting the dataset.
  • Legacy embedding performs well on merchant-driven categories such as dining, health, and transport, but fails on item-purpose distinctions like household vs health or personal care vs health.
  • Harrier zero-shot is stronger on digital_subscriptions, pets, financial_admin, and groceries, but it over-predicts health for many unrelated categories.
  • Neither current embedding approach is enough for robust item-level classification. The next meaningful comparison should be Harrier trained into taxonomy centroids, or Qwen running as a stable local service.

Limitations

  • The benchmark is hand-authored and balanced. It is useful for model comparison, but it is not yet a production distribution.
  • It covers 12 of the 18 main taxonomy categories.
  • It does not yet score subcategory accuracy.
  • It evaluates each item row independently, not full multi-item receipt context.
  • Qwen full benchmark is not included because the current local generation path is too slow for a 120-row interactive run.

Recommended Next Step

The most useful next benchmark is not another zero-shot similarity test. It should be: train taxonomy centroids using Harrier embeddings on labeled item examples, then evaluate both main_category and sub_category.

That would be a fairer embedding-model comparison against the current centroid approach, and it would also be fast enough to run locally.
