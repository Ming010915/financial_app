Item Classification Benchmark Report

This report documents the current receipt item taxonomy and benchmark after replacing the old dataset. It compares the legacy centroid classifier and Harrier on the same 180 item rows, and records the current Qwen local-run status.

Branch: llm-memory Dataset: benchmarks/item_classification_benchmark.csv Taxonomy: taxonomy/receipt_item_taxonomy_summary.json Report date: 2026-07-02

Short Answer

The benchmark has replaced the previous dataset: 180 item rows, 18 main categories, and 146 subcategories.

The full scored comparison currently evaluates main_category only for the two embedding-style approaches. Qwen is still the candidate for item-level main_category plus sub_category, but the local Qwen 4B CLI run did not complete reliably in this interactive run.

Implementation Notes

  • The benchmark runner now supports the compact taxonomy JSON format through sub_categories_by_main.
  • The old taxonomy label digital_subscriptions was replaced with digital_services in executable code paths.
  • Qwen merchant pre-filtering was expanded for current categories such as travel, housing_utilities, digital_services, insurance, and children_family.
  • The Qwen benchmark prompt was compacted to reduce token count, but local generation is still too slow in the current CLI/MPS path.
